# Целевая аудитория

## Цель этапа

Определить кто покупает продукт: сегменты, персоны, их боли и мотивации, где они обитают. Адаптировать вопросы под тип бизнеса (B2B/B2C).

## Входные данные

Прочитай `docs/gtm/context.yaml` — нужна информация о продукте и его типе из предыдущего этапа.

## Вопросы

### Для B2B (type: b2b_saas, services)

1. Какого размера компании — твои клиенты?
   - Стартапы (1-10 человек)
   - Малый бизнес (10-50)
   - Средний бизнес (50-500)
   - Enterprise (500+)
   - Несколько сегментов (уточни)

2. Какие индустрии/вертикали?

3. Кто принимает решение о покупке? (должность, роль)

4. Кто пользуется продуктом? (если отличается от того, кто покупает)

5. Какие у них боли и задачи, связанные с твоим продуктом?

6. Где они ищут решения? Где обитают профессионально?
   - LinkedIn
   - Профессиональные сообщества
   - Конференции
   - Отраслевые медиа
   - Другое

### Для B2C (type: b2c, marketplace)

1. Демография: возраст, пол, география, доход?

2. Чем занимаются? (профессия, образ жизни)

3. Какие интересы связаны с твоим продуктом?

4. Какие боли/желания закрывает продукт?

5. Где они проводят время онлайн?
   - Соцсети (какие)
   - Форумы/сообщества
   - YouTube/TikTok
   - Другое

6. Как они обычно решают эту проблему сейчас?

### Общие вопросы

7. Есть ли уже клиенты/пользователи? Кто они?

8. Кто НЕ является твоей аудиторией? (антиперсона)

## Выходные данные

Запиши в `docs/gtm/context.yaml` секцию `audience`:

```yaml
audience:
  # Для B2B
  company_size: ["startup", "smb"]  # startup | smb | mid | enterprise
  industries:
    - "SaaS"
    - "E-commerce"
  decision_maker: |
    VP of Marketing, CMO.
    Ищет способы оптимизировать процессы.
  end_user: |
    Маркетологи, контент-менеджеры.

  # Для B2C
  demographics:
    age: "25-40"
    gender: "any"
    geography: "US, Europe"
    income: "middle+"

  # Общее
  segments:
    - name: "Ранние последователи"
      description: |
        Технически подкованные пользователи,
        готовые пробовать новое.
      pain_points:
        - "Боль 1"
        - "Боль 2"
      goals:
        - "Цель 1"
      where_they_hang_out:
        - "Twitter"
        - "Product Hunt"

  primary_persona: |
    Детальное описание основной персоны.
    Имя, роль, день из жизни, боли, мотивации.

  anti_persona: |
    Кто точно НЕ наш клиент и почему.

  existing_customers: |
    Описание текущих клиентов если есть.
```

## Критерий завершения

- Определён минимум один сегмент с болями и местами обитания
- Есть описание primary persona
- Пользователь подтвердил что аудитория описана верно
