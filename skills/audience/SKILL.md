# Audience

## Цель

На основе ресёрча предложить типы ЦА. Пользователь валидирует, не генерирует.

## Входные данные

Прочитай `docs/gtm/context.yaml` — нужна информация о продукте и конкурентах.

## Процесс

### 1. Анализ (делаешь сам)

На основе:
- Описания продукта
- Конкурентов и их таргетинга
- Пробелов на рынке
- Обсуждений в Reddit/форумах

Определи:
- Кто реально будет использовать
- Какие сегменты игнорируют конкуренты
- Где самый острый pain

### 2. Дополнительный ресёрч

WebSearch:
- "who uses [категория]"
- "[категория] for [профессия/роль]"
- "[конкурент] customers"
- "[проблема] reddit" — смотри кто жалуется

### 3. Предложи сегменты

Выдай пользователю готовый список:

```
## Целевая аудитория

На основе ресёрча вижу 4 сегмента:

### 1. [Название сегмента]
**Кто:** [описание в 1-2 предложения]
**Размер:** [большой/средний/маленький]
**Pain intensity:** [высокая/средняя/низкая]
**Где обитают:** [платформы, сообщества]
**Почему этот сегмент:** [обоснование]

### 2. [Название сегмента]
...

### 3. [Название сегмента]
...

### 4. [Название сегмента]
...

---

**Мой приоритет:** Начать с сегмента [X] потому что [причина].

Согласен с этими сегментами? Какие убрать/добавить?
```

### 4. Уточнение

Если пользователь корректирует — обнови список. Если добавляет сегмент, которого ты не знаешь — задай 1-2 уточняющих вопроса.

## Выходные данные

### Записать в context.yaml

```yaml
audience:
  segments:
    - name: "Power Users"
      description: |
        Ежедневные пользователи ChatGPT, 5+ разговоров в день.
        Накопили сотни чатов, страдают от потери информации.
      size: "medium"
      pain_intensity: "high"
      where_they_hang_out:
        - "r/ChatGPT"
        - "r/ChatGPTPro"
        - "Twitter AI community"
        - "Hacker News"
      why_this_segment: |
        Самый острый pain, готовы платить за решение,
        активны в сообществах — хороши для early adoption.

    - name: "..."
      ...

  primary_segment: "Power Users"
  primary_segment_rationale: |
    ...
```

## Критерий завершения

- Определены 3-5 сегментов с обоснованием
- Выбран приоритетный сегмент
- Пользователь подтвердил
