# Анализ конкурентов

## Цель этапа

Найти и проанализировать конкурентов, понять их позиционирование, найти свободные ниши. Использовать веб-поиск активно.

## Входные данные

Прочитай `docs/gtm/context.yaml` — нужна информация о продукте и аудитории для поиска конкурентов.

## Процесс

### 1. Сбор информации от пользователя

1. Каких конкурентов ты уже знаешь? (прямые и непрямые)

2. Какие ключевые слова описывают твой продукт? (для поиска)

3. Как люди решают проблему сейчас, без специального инструмента?

### 2. Активный веб-поиск

Используй WebSearch для поиска конкурентов:
- Поиск по ключевым словам продукта
- "[категория] tools/software/apps"
- "alternatives to [известный конкурент]"
- "best [категория] for [аудитория]"

Используй WebFetch для анализа сайтов конкурентов:
- Главная страница — позиционирование, tagline
- Pricing — модель, цены, сегменты
- Features — функционал

### 3. Анализ

Для каждого конкурента определи:
- Позиционирование (для кого, какую проблему решает)
- Ценовой сегмент
- Сильные стороны
- Слабые стороны/пробелы
- Размер (если можно оценить)

### 4. Карта и позиционирование

Построй карту конкурентов по осям (выбери релевантные):
- Цена (дёшево — дорого)
- Сложность (просто — сложно)
- Аудитория (SMB — Enterprise)
- Функционал (узко — широко)

Найди:
- Где скопление конкурентов (красный океан)
- Где пусто (возможные ниши)
- Где позиционируется продукт пользователя

## Выходные данные

### docs/gtm/context.yaml — секция competitors:

```yaml
competitors:
  direct:
    - name: "Конкурент 1"
      url: "https://..."
      positioning: "Для X, решает Y"
      pricing: "$XX/mo, freemium"
      strengths:
        - "Сильная сторона"
      weaknesses:
        - "Слабая сторона"
      size: "large"  # small | medium | large | unknown

  indirect:
    - name: "Excel/Spreadsheets"
      description: |
        Как люди решают проблему без специального инструмента.

  substitutes:
    - name: "Агентства"
      description: |
        Альтернативный способ решения — аутсорс.

  positioning_map:
    axes:
      x: "Цена (low → high)"
      y: "Сложность (simple → complex)"
    our_position: "low price, simple"
    insights: |
      Большинство конкурентов в enterprise сегменте.
      Ниша простых решений для SMB свободна.

  our_differentiation: |
    Детальное описание как мы отличаемся
    от каждого типа конкурентов.

  competitive_advantages:
    - "Преимущество 1"
    - "Преимущество 2"
```

### docs/gtm/research/competitors.md

Сохрани детальный анализ каждого конкурента:
- Скриншоты позиционирования (описания)
- Детали pricing
- Заметки по функционалу
- Источники информации

## Критерий завершения

- Проанализировано минимум 3-5 прямых конкурентов
- Определены непрямые конкуренты/заменители
- Построена карта позиционирования
- Сформулированы конкурентные преимущества
- Пользователь подтвердил анализ
