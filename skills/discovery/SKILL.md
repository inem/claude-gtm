# Discovery

## Цель

Понять что за продукт и провести глубокий ресёрч рынка. Минимум вопросов, максимум самостоятельного анализа.

## Вопросы пользователю (только эти)

Задай эти вопросы одним сообщением:

1. **Что за продукт?** Опиши в 2-3 предложениях что он делает.

2. **Какая стадия?**
   - Идея
   - Есть MVP/прототип
   - Есть пользователи
   - Есть платящие

3. **Есть конкуренты, которых уже знаешь?** (опционально)

**Всё.** Остальное узнаёшь сам.

## Самостоятельный ресёрч

После ответов пользователя:

### 1. Поиск конкурентов

Используй WebSearch:
- "[категория продукта] tools"
- "[категория] app/extension/software"
- "best [категория] for [use case]"
- "alternatives to [известный конкурент]"
- "[проблема] solution"

### 2. Анализ конкурентов

Для каждого найденного (5-10 штук) используй WebFetch:
- Главная страница — positioning, tagline, messaging
- Pricing — модель, сегменты
- Features — что делают, чего не делают

Записывай:
- Как формулируют проблему
- На кого таргетятся
- Ценовой сегмент
- Сильные/слабые стороны

### 3. Поиск отзывов и обсуждений

WebSearch:
- "[конкурент] review"
- "[конкурент] reddit"
- "[категория] reddit recommendations"
- "why I stopped using [конкурент]"

Ищи:
- На что жалуются
- Чего не хватает
- Почему уходят

### 4. Тренды

WebSearch:
- "[категория] trends 2024"
- "future of [категория]"

## Выходные данные

### Показать пользователю

После ресёрча выдай summary:

```
## Что я нашёл

### Рынок
- Категория: [название]
- Размер: [оценка если есть]
- Тренд: [растёт/падает/стабильно]

### Конкуренты (топ 5)

| Название | Positioning | Цена | Сильные стороны | Слабые стороны |
|----------|-------------|------|-----------------|----------------|
| ... | ... | ... | ... | ... |

### Пробелы на рынке
1. [Пробел 1 — никто не делает X]
2. [Пробел 2 — все игнорируют сегмент Y]
3. ...

### Где твой продукт
[Как продукт вписывается в карту, какой пробел закрывает]

---

Это соответствует твоему видению? Что поправить?
```

### Записать в context.yaml

```yaml
product:
  name: "..."
  description: |
    ...
  stage: "mvp"

market:
  category: "..."
  trends:
    - "..."

competitors:
  - name: "..."
    url: "..."
    positioning: "..."
    pricing: "..."
    strengths:
      - "..."
    weaknesses:
      - "..."

gaps:
  - description: "..."
    opportunity: "..."
```

### Записать в docs/gtm/research/competitors.md

Детальный анализ каждого конкурента с цитатами, скриншотами описаний, источниками.

## Критерий завершения

- Проанализировано 5-10 конкурентов
- Найдены пробелы на рынке
- Пользователь подтвердил что картина верная
