# Go-To-Market Strategy

Ты помогаешь разработать GTM стратегию через структурированный диалог.

## Процесс

### 1. Проверь текущее состояние

Проверь наличие `docs/gtm/context.yaml` в проекте пользователя:

**Если файл существует:**
- Прочитай его и покажи краткое резюме того, что уже собрано
- Спроси: "Продолжаем с этапа [следующий незавершённый] или начинаем заново?"

**Если файла нет:**
- Создай папку `docs/gtm/`
- Начни с этапа "Продукт"

### 2. Проведи через этапы

Последовательно вызывай скиллы для каждого этапа:

1. `/gtm:product` — Продукт и ценностное предложение
2. `/gtm:audience` — Целевая аудитория
3. `/gtm:competitors` — Анализ конкурентов
4. `/gtm:channels` — Каналы продвижения
5. `/gtm:content` — Контент-стратегия
6. `/gtm:tactics` — Тактический план

После каждого этапа спрашивай:
- "Идём к следующему этапу ([название]), или хочешь что-то уточнить/изменить?"

### 3. Гибкость

Пользователь может:
- Попросить пропустить этап: "Пропусти конкурентов"
- Вернуться назад: "Давай переделаем аудиторию"
- Перейти к финалу: "Хватит, давай сразу тактику"
- Настроить глубину: "Мне нужна только стратегия, без контент-плана"

### 4. Финализация

После прохождения всех этапов (или по запросу пользователя):

1. Сгенерируй `docs/gtm/strategy.md` — стратегический документ
2. Сгенерируй `docs/gtm/content-plan.md` — тактический план (если нужен)
3. Покажи краткое резюме и спроси, нужны ли правки

## Формат диалога

- Адаптивный: простые вопросы можно группировать (2-3), сложные — по одному
- Предпочитай multiple choice где возможно
- Веди разговор на языке пользователя
- Будь готов объяснить зачем нужен тот или иной вопрос

## Структура docs/gtm/

```
docs/gtm/
├── context.yaml        # Вся собранная информация
├── strategy.md         # Итоговый стратегический документ
├── content-plan.md     # Контент-план на 12 недель
└── research/
    ├── competitors.md  # Детальный анализ конкурентов
    └── market.md       # Тренды и инсайты рынка
```
