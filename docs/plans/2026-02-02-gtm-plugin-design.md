# GTM Plugin Design

Claude Code плагин для разработки Go-To-Market стратегии через структурированный диалог.

## Обзор

Плагин проводит пользователя через этапы разработки GTM стратегии:
1. Задаёт вопросы, собирает информацию о продукте
2. Активно ресёрчит рынок и конкурентов через веб-поиск
3. Генерирует стратегический документ и тактический план

## Архитектура

**Подход:** Линейный pipeline с возможностью вызова отдельных этапов.

### Структура плагина

```
claude-gtm/
├── .claude-plugin/
│   └── plugin.json
├── skills/
│   ├── gtm.md              # Главный скилл — оркестратор
│   ├── gtm-product.md      # Этап 1: Продукт и ценность
│   ├── gtm-audience.md     # Этап 2: Аудитория и сегменты
│   ├── gtm-competitors.md  # Этап 3: Конкуренты и позиционирование
│   ├── gtm-channels.md     # Этап 4: Каналы дистрибуции
│   ├── gtm-content.md      # Этап 5: Контент-стратегия
│   └── gtm-tactics.md      # Этап 6: Тактический план
└── README.md
```

### Хранение данных

Результаты сохраняются в `docs/gtm/` проекта пользователя:

```
docs/gtm/
├── context.yaml            # Собранная информация (машиночитаемый)
├── strategy.md             # Итоговый стратегический документ
├── content-plan.md         # Контент-план и календарь
└── research/
    ├── competitors.md      # Результаты анализа конкурентов
    └── market.md           # Тренды и инсайты рынка
```

### Формат context.yaml

```yaml
product:
  name: "МойПродукт"
  problem: |
    Многострочное описание проблемы
    которую решает продукт.
  value_proposition: |
    Ценностное предложение
    в нескольких строках.
  type: "b2b_saas"

audience:
  segments:
    - name: "Стартапы"
      size: "medium"
      pain_points:
        - "Боль 1"
        - "Боль 2"
  primary_persona: |
    Описание основной персоны.

competitors:
  direct:
    - name: "Конкурент 1"
      url: "https://..."
      positioning: "..."
  positioning: |
    Наше позиционирование
    относительно конкурентов.

channels:
  primary:
    - name: "Content Marketing"
      priority: 1
      rationale: "..."
  secondary:
    - name: "LinkedIn"
      priority: 2

content:
  topics: [...]
  formats: [...]
  tone_of_voice: "..."

tactics:
  timeline_weeks: 12
  budget: "optional"
```

## Скиллы

### Главный скилл `/gtm`

Оркестратор, который:
1. Проверяет наличие `docs/gtm/context.yaml`
2. Если есть — показывает прогресс, предлагает продолжить или начать заново
3. Если нет — создаёт папку и начинает с этапа "Продукт"
4. После каждого этапа спрашивает: "Идём дальше или хочешь что-то уточнить?"
5. В конце генерирует итоговые документы

### Формат скилла-этапа

```markdown
---
name: gtm-product
description: Этап 1 — анализ продукта и ценностного предложения
---

## Цель этапа
Собрать информацию о продукте, проблеме, решении и определить тип бизнеса.

## Входные данные
- Читает `docs/gtm/context.yaml` (если есть предыдущий контекст)

## Вопросы для пользователя
1. Как называется продукт и что он делает? (открытый)
2. Какую проблему решает? (открытый)
3. Тип продукта: [B2B SaaS / B2C приложение / Услуги / Маркетплейс / Другое]
4. Стадия: [Есть платящие / Есть бесплатные / Только идея]
5. Чем отличаетесь от существующих решений? (открытый)

## Выходные данные
Записать в `context.yaml` секцию `product:`

## Критерий завершения
Секция `product` заполнена, пользователь подтвердил.
```

### Этапы

| Этап | Скилл | Что делает |
|------|-------|------------|
| 1 | gtm-product | Продукт, проблема, решение, тип бизнеса, отличия |
| 2 | gtm-audience | Сегменты, персоны, боли, мотивации, где обитают |
| 3 | gtm-competitors | Веб-поиск конкурентов, анализ, карта, ниши |
| 4 | gtm-channels | Каналы дистрибуции, приоритизация |
| 5 | gtm-content | Темы, форматы, tone of voice, рубрикатор |
| 6 | gtm-tactics | Контент-план 12 недель, задачи, метрики |

## Установка и использование

**Установка:**
```
/install-plugin github:inem/claude-gtm
```

**Использование:**
```
/gtm                  — полный цикл
/gtm:product          — только анализ продукта
/gtm:audience         — только аудитория
/gtm:competitors      — только конкуренты
/gtm:channels         — только каналы
/gtm:content          — только контент-стратегия
/gtm:tactics          — только тактический план
```

## Принципы диалога

- Адаптивный формат вопросов: простые группировать, сложные по одному
- Предпочитать multiple choice где возможно
- Активный веб-поиск для анализа конкурентов и рынка
- Возможность пропустить этап или вернуться назад
- Пользователь может настроить глубину проработки
